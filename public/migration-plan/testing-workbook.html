Yarny React Migration · Manual Testing Workbook

This workbook captures the manual validation required as we finalize the React migration. These pages focus on scenarios that require human judgment, visual inspection, and UX feedback.

Work through the phases in order. Each page clusters related sections so you can stay focused on a single theme without scrolling through thousands of lines.

JSON Primary Architecture (✅ Complete):

  • ✅ Fast saves: Snippet content saves to JSON files first (<50ms), then syncs to Google Docs in the background.
  • ✅ Background sync: Service Worker + Background Sync API ensures changes persist even if the page is closed.
  • ✅ Conflict detection: Automatically detects when Google Docs are modified externally and prompts for resolution.
  • ✅ Sync status: Editor footer shows sync status indicator (Synced/Syncing/Pending/Failed).
  • ✅ Migration system: One-time migration of existing stories from Google Docs to JSON files.

Testing note: When testing save operations, verify that saves complete quickly (<50ms) and check the sync status indicator to confirm background sync to Google Docs is working.

Shared Test Corpus: Use the Yarny Test Corpus (small / medium / large) on Google Drive. Start with test-small for sanity checks, then expand to medium and large when validating performance and sync behavior.

Yarny React Migration - Testing Workbook

This workbook guides you through testing every feature and function after the React migration is complete. Check off each item as you test it, and note any issues you encounter.

Note: This workbook is designed for non-developers. Each test includes clear instructions and expected results.

Test Corpus: For comprehensive testing, use the "Yarny Test Corpus" folder in Google Drive, which contains three project sizes (small, medium, large) with sample data. Start with the small project for basic tests, then use medium and large projects for performance and edge case testing.

lock Section 1: Authentication & Access

Test 1.1: Google Sign-In

Verify that you can sign in with your Google account.

  1. Go to the login page (yarny.lindsaybrunner.com/react)
  2. Click "Sign in with Google"
  3. Select your Google account
  4. Authorize Yarny to access your account

Test 1.2: Google Drive Connection

Verify that you can connect your Google Drive account.

  1. After signing in, you should be prompted to connect Google Drive
  2. Click "Connect to Drive" or similar button
  3. Authorize Yarny to access your Google Drive
  4. Verify you see the stories list after connection

Test 1.3: Session Persistence

Verify that your session persists when you refresh the page or close and reopen the browser.

  1. Sign in and connect Drive
  2. Refresh the page (F5 or Cmd+R)
  3. Verify you remain signed in (don't need to sign in again)
  4. Close the browser tab and reopen it
  5. Navigate back to yarny.lindsaybrunner.com/react
  6. Verify you remain signed in (session should last 48 hours)

Test 1.4: Sign Out

Verify that you can sign out properly.

  1. Click "Sign out" button (usually in header or footer)
  2. Verify you are redirected to login page
  3. Verify you need to sign in again to access stories

Test 3.5: Plain Text Only (No Rich Formatting)

Verify that the editor only supports plain text (no bold, italic, colors, etc.) and preserves paragraph/line breaks correctly.

  1. Type some text in the editor
  2. Try to make text bold (Ctrl+B or Cmd+B) - it should NOT work
  3. Try to make text italic (Ctrl+I or Cmd+I) - it should NOT work
  4. Paste rich text from another document
  5. Verify the formatting is stripped and only plain text remains
  6. Test paragraph breaks: Press Enter twice to create a paragraph break
  7. Verify paragraph breaks (double Enter) create proper spacing between paragraphs
  8. Test soft line breaks: Press Shift+Enter to create a line break within a paragraph
  9. Verify single line breaks (Shift+Enter) work correctly without creating new paragraphs
  10. Save the snippet and open it in Google Docs
  11. Verify paragraph breaks and line breaks are preserved correctly in Google Docs

Test 3.6: Editor Authority (Editor is Truth While Open)

Verify that while a snippet is open in the editor, the editor is authoritative and external changes in Google Docs are ignored until you switch snippets.

  1. Open a snippet in the Yarny editor
  2. Type some text in the editor
  3. In another tab, open the same snippet's Google Doc in Google Docs
  4. Make changes in Google Docs and save
  5. Return to Yarny (don't switch snippets yet)
  6. Verify your Yarny edits are still visible (editor is authoritative)
  7. Continue editing in Yarny and save
  8. Verify your Yarny changes overwrite the Google Docs changes
  9. Open the Google Doc again
  10. Verify it matches your Yarny version (not the Google Docs edits you made earlier)
